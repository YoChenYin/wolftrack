# Trend Core 核心功能 — 實作邏輯補完版

> 本文件把濃縮版 spec 中「有描述但沒定義計算方式」的部分補齊。
> 標示 **⚠️假設** 的地方，是我用業界常見的量化趨勢跟蹤方法推導出的合理預設值，
> 不是從原網站反推出的真實邏輯，需要你們與資料/量化團隊核對後才能當正式規格。
> 標示 **✅可反推** 的地方，是可以從網站行為/文案直接還原、把握度較高的邏輯。

## 1. Core Score（核心分數）

**用途**：同板塊內排名股票強弱的綜合分數，0–100。

**⚠️假設：建議組成（需與量化團隊核對權重）**

| 因子 | 說明 | 建議權重 |
|---|---|---|
| 均線排列 | 價格 vs 20/50/200 日均線的相對位置與均線多頭排列程度 | 30% |
| 動能 | ROC(20)、RSI(14) 是否在強勢區間（RSI 50–70 為佳，>80 視為過熱扣分） | 25% |
| 趨勢強度 | ADX(14)，>25 代表趨勢明確，數值越高分數越高 | 20% |
| 相對強度 | 該股近 20/60 日報酬率 相對 板塊指數 / SPY 的超額報酬 | 15% |
| 量能確認 | 近 5 日均量 vs 近 20 日均量的放大比例（趨勢延續需要量能佐證） | 10% |

**公式骨架：**
```
Core Score = 0.30 × MA_Score + 0.25 × Momentum_Score
           + 0.20 × ADX_Score + 0.15 × RelStrength_Score
           + 0.10 × Volume_Score
（每個子分數先各自正規化到 0–100 再加權）
```

**輸出**：每日收盤後批次計算一次，隔日開盤前更新。

## 2. 三段式趨勢分類邏輯（最核心的判斷）

**⚠️假設**：以下門檻為常見趨勢跟蹤策略的合理起點，正式上線前務必用歷史資料回測校準。

### 2.1 🔵 趨勢雷達（反轉 Reversal）
判斷「剛要變盤」：
- 短均線（如 20MA）由下往上（或由上往下）穿越中均線（如 50MA），且穿越發生在**近 5 個交易日內**
- MACD 柱狀圖由負轉正（或正轉負），同樣限定近 5 日內
- 當日或近 3 日內出現量能放大（成交量 > 20 日均量 1.5 倍）作為確認訊號，避免假突破
- **反轉點日期** = 均線黃金/死亡交叉發生的實際日期

```
IF (MA20 cross above MA50 within last 5 days)
   AND (MACD_hist turns positive within last 5 days)
   AND (volume_spike_confirmed = true)
THEN status = "反轉雷達", reversal_point_date = cross_date
```

### 2.2 🟡 蓄勢待發（回檔 Pullback）
判斷「上升趨勢中的健康回檔，等拉回上車」：
- 大趨勢仍成立：MA20 > MA50 > MA200（多頭排列未破壞）
- 價格從近期高點回落 **5%–15%**（超過15%視為趨勢轉弱，不算蓄勢待發）
- 價格回測到支撐位（如 MA20 或 MA50 附近 ±2%）但未跌破
- RSI 從超買區（>70）降溫至 40–55 區間（動能冷卻但未轉空）

```
IF (MA20 > MA50 > MA200)
   AND (drawdown_from_recent_high BETWEEN 5% AND 15%)
   AND (price within ±2% of MA20 or MA50)
   AND (RSI cooled from >70 to 40–55 range)
THEN status = "蓄勢待發"
```

### 2.3 🟢 趨勢穩健（Bullish Continuation）
判斷「趨勢已經在跑，動能延續中」：
- 多頭排列：MA20 > MA50 > MA200，且三條均線斜率皆向上
- ADX(14) > 25 且呈上升趨勢（代表趨勢動能未衰退）
- 價格持續創近期新高（如近 20 日內出現新高 ≥ 2 次）
- 未出現顯著回檔（drawdown_from_recent_high < 5%）

```
IF (MA20 > MA50 > MA200, all slopes rising)
   AND (ADX > 25 and rising)
   AND (new_high_count_last_20d >= 2)
   AND (drawdown_from_recent_high < 5%)
THEN status = "趨勢穩健"
```

> **互斥規則**：一檔股票每日只能落在一種狀態；判斷順序建議「反轉 → 蓄勢待發 → 趨勢穩健」，符合前者條件就不再檢查後者，避免同時符合多個條件造成分類衝突。

## 3. ✅可反推：趨勢變化後漲跌幅

這是唯一原本就邏輯清楚的功能，正式定義如下：

```
change_pct_since_signal = (price_now - price_at_signal_date) / price_at_signal_date × 100%
```
- `price_at_signal_date`：訊號觸發當日收盤價（例如反轉點那天）
- `price_now`：最新收盤價
- 用途：讓使用者一眼判斷「這個訊號歷史上跟到現在，賺了還是賠了」
- 建議同時顯示「距今交易日數」，避免漲跌幅脫離時間脈絡（例如 +30% 但已經過了 200 天，跟 +30% 只花 10 天意義完全不同）

## 4. 內部人交易訊號分級

**⚠️假設**：

| 分級 | 條件 |
|---|---|
| 高關注 | 近 30 天內，CEO 或 CFO 本人買入，且單筆金額 > $500K，或連續買入 ≥ 2 次 |
| 中關注 | 近 30 天內，董事會成員買入，金額 > $100K |
| 一般 | 其他 Form 4 申報買入紀錄 |

- 排除賣出（賣出常因課稅、期權行使等非訊號性原因，建議預設過濾掉，僅保留買入）
- 排除授予/獎酬型的自動交易（Rule 10b5-1 計畫內的定期交易雜訊需標記或過濾，這是「自動過濾雜訊」文案背後應有的邏輯）

## 5. 機構持股異動：兩個關鍵偵測邏輯

### 5.1 狼群效應（Wolf Pack）
**⚠️假設**：
```
IF (distinct_institutions_filing_13D/13G_buy(ticker) >= 3)
   AND (all filings occurred within trailing 30 days)
THEN flag = "狼群效應"
```
- 門檻值（3家機構 / 30天）需依實際 13D/13G 申報頻率調整，避免過度觸發或太少觸發

### 5.2 階梯加倉（Ladder Buying）
**⚠️假設**：
```
IF (same_institution_filed_increase(ticker) >= 3 times)
   AND (all within trailing 90 days)
   AND (each filing shows stake % increased vs previous filing)
THEN flag = "階梯加倉"
```

## 6. 趨勢王者重現（Proven Winners Reappearing）

**⚠️假設 — 這個模組邏輯把握度最低，務必跟產品端確認**：

概念：找出「歷史上出現過某訊號組合後，績效顯著跑贏大盤」的股票，當**同一檔股票**再次出現**同樣的訊號組合**時標記出來。

```
Step 1（歷史建檔）：
  對每檔股票掃描歷史所有「反轉雷達」訊號事件
  計算：訊號後 20/60 日的報酬率
  標記「歷史高績效事件」= 訊號後 20 日報酬率 > 板塊平均 + 1 個標準差

Step 2（現在比對）：
  若某股票「過去曾有 ≥1 次歷史高績效事件」
  且「今天再次觸發反轉雷達訊號」
  → 標記為「趨勢王者重現」，並附帶顯示歷史那次事件的報酬率作為參考
```

- 建議附加「歷史勝率」欄位：該股票過去所有反轉訊號中，正報酬的比例
- 這個功能本質是「用個股自己的歷史訊號成功率，做二次篩選」，需要至少 2–3 年历史資料才有統計意義

## 7. Trend Core 選股（歷史回測）— 4 種模式的進出場規則

**⚠️假設**：

| 模式 | 進場條件 | 出場條件 |
|---|---|---|
| 雙向趨勢 | MA20 上穿 MA50 做多；MA20 下穿 MA50 做空 | 反向訊號出現時平倉反手 |
| 多頭 | 僅 MA20 上穿 MA50 時做多 | MA20 下穿 MA50 或 跌破 MA50 停損 |
| 空頭 | 僅 MA20 下穿 MA50 時放空 | MA20 上穿 MA50 或 突破 MA50 停損 |
| 逢低買進（Buy the Dip） | 大趨勢多頭（MA50 > MA200）且回檔至 MA20 支撐時進場 | 跌破 MA50 停損，或達到目標報酬率停利 |

**回測輸出欄位**（建議）：
- 總報酬率 / 年化報酬率
- 勝率、平均獲利/平均虧損
- 最大回撤（MDD）
- 交易次數、平均持有天數
- 逐筆交易明細（進出場日期、價格、損益）

## 8. 資料更新與時區

**⚠️假設**：
- 計算時機：美股收盤後（ET 16:00 收盤 → 建議 ET 18:00–20:00 完成批次計算）
- 顯示時區：介面統一轉換為使用者所在時區，另需標示原始資料時區（ET）避免混淆
- 歷史資料回補：新股票加入資料庫時，至少回補 2 年日線資料才能計算 200MA、趨勢王者比對等長週期指標
