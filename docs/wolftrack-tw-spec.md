# WolfTrack TW（狼蹤台股版）— 開發規格書

> 基礎架構沿用美股版（三段式趨勢分類 + Core Score + 訊號後漲跌幅），
> 核心差異：加入台股特有的「三大法人籌碼」與「券商分點」資料，
> 這是台股散戶最愛看、但美股沒有對應概念的獨特優勢，作為本版本的護城河。

---

## 一、台股 vs 美股版本的關鍵差異

| 面向 | 美股版 | 台股版 |
|---|---|---|
| 內幕/主力訊號來源 | 內部人交易 (Form 4)、機構 13D/13G | **三大法人買賣超**（外資/投信/自營商）、**券商分點進出** |
| 漲跌幅限制 | 無限制 | ±10%（需納入判斷，漲跌停會扭曲技術指標） |
| 除權息 | 較少影響 | **必須做還原股價**，否則均線/趨勢判斷會在除權息當天失真 |
| 交易時間 | 4小時+盤前盤後 | 09:00–13:30，無盤前盤後 |
| 資料來源 | Polygon/FMP 等付費 API | **證交所 OpenAPI（TWSE）**、**櫃買中心 OpenAPI（TPEx）**、公開資訊觀測站(MOPS) |
| 融資融券 | 較不常用作訊號 | **融資維持率、借券賣出餘額**是重要風險/軋空訊號 |

## 二、必留核心：板塊重點股 × 三段式趨勢分類（沿用美股版邏輯）

分類邏輯（反轉雷達 / 蓄勢待發 / 趨勢穩健）與 Core Score 的技術面計算（均線排列、ADX、RSI、量能）**架構不變**，但有兩個台股必要調整：

1. **還原股價**：所有均線、漲跌幅計算前，股價需先做除權息還原，否則除息當天會被誤判為「反轉雷達」（假跌破均線）
2. **漲跌停過濾**：當日觸及漲跌停（±10%）時，量能/動能指標會失真，需標記為「特殊狀態」而非直接套用一般分類邏輯，避免誤判

## 三、台股版最核心的差異化功能：籌碼分數（Chip Score）

**這是台股版取代/補強美股版「內部人交易+機構持股」的核心模組，權重最高，務必優先開發。**

### 3.1 三大法人買賣超（Institutional Trading）

資料來源：TWSE / TPEx 每日三大法人買賣金額表

| 法人類別 | 訊號權重（⚠️假設） | 說明 |
|---|---|---|
| 投信 | 高（40%） | 資金集中、選股精準，是台股「認養股」的主要推手，訊號意義最強 |
| 外資 | 中高（35%） | 資金量體最大，趨勢延續性強，但單日買賣超易受程式交易干擾 |
| 自營商 | 低（10%） | 常做避險/當沖，雜訊多，僅作輔助參考 |
| 三大法人合計 | 加分項（15%） | 三者同步買超時給予額外加分 |

**籌碼分數公式骨架：**
```
Chip_Score = 0.40 × InvestTrust_Score + 0.35 × Foreign_Score
           + 0.10 × Dealer_Score + 0.15 × Alignment_Bonus

InvestTrust_Score = f(近5日投信買超張數, 近20日投信買賣超趨勢)
Foreign_Score = f(近5日外資買超金額, 外資持股比例變化)
Alignment_Bonus = 三大法人是否同方向買超（同買/同賣）
```

**最終 Core Score 需調整為納入籌碼面：**
```
Core Score (台股版) = 0.50 × Technical_Score（原美股版5因子加總）
                     + 0.50 × Chip_Score
```
> ⚠️假設：技術面與籌碼面各佔一半，因為籌碼是台股版的核心賣點，權重刻意拉高於美股版（美股版機構訊號僅作輔助驗證，非直接進 Core Score），需與量化團隊核對是否要調整比例。

---

### 3.2 投信認養偵測（Investment Trust Adoption）— 台股版「狼群效應」

台股經典模式：投信一旦鎖定一檔中小型股連續買超，股價常有波段行情，是台股散戶最愛跟的訊號之一。

```
IF (investment_trust_net_buy_days_consecutive(ticker) >= 3)
   AND (investment_trust_holding_pct_increased = true)
   AND (market_cap < 中大型股門檻，如 300億)  # 投信認養在中小型股更明顯
THEN flag = "投信認養中"
```

**⚠️假設**：連續買超天數門檻（3天）、市值門檻（300億）需回測校準。

---

### 3.3 外資階梯加碼（Foreign Ladder Buying）— 對應美股版「階梯加倉」

```
IF (foreign_holding_pct increased for >=3 consecutive weeks)
   AND (each week's increase > 前一週)
THEN flag = "外資階梯加碼"
```

---

### 3.4 券商分點主力進出（Broker Branch Flow）— 台股獨有，美股沒有對應功能

台股特有公開資料：可查詢**個股在哪些券商分點買賣超**，用來偵測主力/大戶動向，是台股當沖/波段客最常用的籌碼工具之一，強烈建議做為 WolfTrack TW 的招牌功能。

```
IF (單一券商分點於近5日買超金額 排名該股當日買超前3名)
   AND (該分點過去30日對同一股票有多次買超紀錄)
THEN flag = "主力券商進駐"，並列出分點名稱與買超張數
```

**資料來源**：各券商/看盤軟體常見的「分點進出」資料，需另外確認資料供應商（台股較無官方公開 API，多數來自第三方資料商如 CMoney、嘉實 XQ、Fugle 等，需談資料授權）

---

### 3.5 融資融券與借券賣出（風險/軋空訊號）

**⚠️假設**：

| 訊號 | 條件 | 意義 |
|---|---|---|
| 融資使用率過高警示 | 融資餘額 / 股本 > 歷史均值 + 1標準差 | 籌碼浮動大，回檔風險高 |
| 融資維持率偏低警示 | 接近追繳水位（一般 130% 附近） | 有斷頭賣壓風險 |
| 借券賣出異常增加 | 借券賣出餘額近5日增幅 > 50% | 空方力道增強，或潛在軋空題材 |

### 3.6 籌碼集中度指標（Chip Concentration Index）— 取自 AlphaFlow TW 既有邏輯，作為進出場訊號

> 來源：你在 AlphaFlow 台股版已實作「5日/10日/20日 外資投信籌碼集中度」，這個邏輯比 3.1 的籌碼分數更進一步——3.1 是「今天籌碼夠不夠強」的靜態分數，3.6 是「籌碼力道正在增強還是減弱」的**動態進出場訊號**，兩者互補，建議直接沿用你原本的邏輯併入 WolfTrack TW。

**定義**（已確認：以張數計算，非金額）：
```
Concentration(N) = (近N日外資買超張數 + 近N日投信買超張數) / 近N日總成交量（張） × 100%
```
分別計算 `Concentration(5)`、`Concentration(10)`、`Concentration(20)` 三個窗口。

**進場訊號（籌碼多頭排列）**：
```
IF Concentration(5) > Concentration(10) > Concentration(20)
   AND Concentration(5) > 0
THEN chip_momentum = "轉強"（類似均線多頭排列，但看的是籌碼而非價格）
```
意義：短期籌碼集中度領先中長期，代表外資投信買超力道正在**加速**，比單看「今天有沒有買超」更能抓到主升段起漲點。

**出場訊號（籌碼排列反轉）**：
```
IF Concentration(5) < Concentration(10)
   （短期集中度開始弱於中期，代表買盤力道轉弱/開始獲利了結）
THEN chip_momentum = "轉弱"，建議標記為出場警示
```

**與 Core Score 的關係**：
- 3.1 的 Chip_Score 是**分數**，用於同板塊排名比較
- 3.6 的 chip_momentum 是**方向訊號**（轉強/持平/轉弱），直接作為三段式分類（反轉雷達/蓄勢待發/趨勢穩健）額外的確認條件：
```
IF (符合「趨勢穩健」技術面條件) AND (chip_momentum = "轉強")
THEN 分類信心度提升，UI 上額外標記「籌碼確認」徽章

IF (符合「趨勢穩健」技術面條件) BUT (chip_momentum = "轉弱")
THEN 分類降級為「蓄勢待發」或標記「籌碼背離」警示
   （價格還在漲，但法人籌碼已經在退，是台股常見的出貨前兆）
```

**✅已確認**：計算單位為張數，公式與 AlphaFlow 原網站邏輯一致。

## 四、供應鏈估值比較（新模組）— 取自 AlphaFlow TW 既有邏輯

> 這個模組跟前面所有「趨勢/籌碼」邏輯性質不同，是**價值面 + 族群輪動**的篩選器，用來找「同一條供應鏈裡，漲得慢、估值還便宜」的落後股，是台股很經典的「補漲」交易邏輯，建議獨立成一個模組，不要硬塞進 Core Score。

### 4.1 核心邏輯

```
Step 1：定義供應鏈/族群分組（如：AI伺服器供應鏈、CoWoS封裝、PCB上游等）
Step 2：計算族群內每檔股票的 PE、PB
Step 3：算出族群的 PE/PB 中位數（Group_Median_PE, Group_Median_PB）
Step 4：計算每檔股票在族群內的估值百分位（Valuation_Percentile）
        → 百分位越低 = 相對族群其他成員越便宜
Step 5：計算每檔股票近20日股價漲幅，並與族群平均漲幅比較
```

### 4.2 落後股篩選條件

```
IF (Valuation_Percentile <= 30%，即族群內估值後段班)
   AND (族群平均近20日漲幅 > 大盤平均漲幅，代表這條供應鏈題材正熱)
   AND (該股自己近20日漲幅 < 族群平均漲幅，代表還沒補漲)
THEN flag = "供應鏈落後股"，列入補漲觀察名單
```

**輸出欄位建議**：
- `group_name`（所屬供應鏈/族群）
- `pe_percentile_in_group` / `pb_percentile_in_group`
- `group_avg_return_20d` vs `stock_return_20d`（漲幅落差，數字越大代表補漲空間理論上越大）
- `peer_list`（同族群其他成員，方便使用者直接比較）

### 4.3 與既有模組的搭配使用

這個模組**不建議**跟三段式趨勢分類/Core Score 混在一起計算，而是作為一個**獨立篩選器**放在個股詳情頁或另開一個「族群估值比較」頁籤：
- 使用情境：使用者先在三欄戰術面板看到某檔「趨勢穩健」的熱門股 → 點進去 → 系統自動帶出「同族群還有哪些股票估值偏低、還沒補漲」→ 形成「找龍頭 → 找還沒漲的關聯股」的完整動線
- 這也是台股散戶很習慣的操作邏輯（跟龍頭找補漲股），建議做成龍頭股詳情頁的一個 side panel，而不是獨立入口，使用率會更高

**✅已確認資料維護方式**：供應鏈/族群分組不走即時 API，而是**整理靜態分類文章、寫進 config 檔案**（一次性建置、之後手動維護增補），不需要每次跑爬蟲。來源：[MY-Learning 理財通《台股概念股有哪些》](https://www.my-finance.com.tw/tw/News_detail/2285/台股概念股有哪些) 整理的產業概念股／集團概念股分類架構。

## 五、沿用美股版但需微調的功能

### 5.1 趨勢王者重現
邏輯不變（歷史高績效訊號重現），但**比對條件建議加入籌碼面**：
```
歷史高績效事件定義 = 訊號後20日報酬率 > 板塊均值+1標準差
                     AND 當時投信/外資為淨買超
```
> 台股波段行情高度仰賴法人籌碼，單純技術面重現的參考價值低於美股，建議必須疊加籌碼條件才算「真正的王者重現」。

### 5.2 Trend Core 選股（歷史回測）
四種模式（雙向趨勢/多頭/空頭/逢低買進）邏輯不變，但**回測時需排除除權息日**造成的價格跳空，且**放空模式需檢查該股是否可融券**（台股非所有個股皆可放空，需對照可借券/融券餘額資料）。

### 5.3 訊號後漲跌幅
公式不變，但需用**還原股價**計算，否則除權息會讓漲跌幅失真。
