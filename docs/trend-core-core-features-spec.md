# Trend Core 核心功能規格（濃縮版）
> 設計目標：交易效率與精準度優先，只留下能直接影響選股/進出場決策的功能

---

## 一、必留核心：板塊重點股 × 三段式趨勢分類

**這是整個系統的心臟，一定要保留且做到最快、最準。**

每天對全市場（2000+ 檔）依板塊/產業分組，並將每檔股票即時歸類到以下三種戰術狀態之一：

| 分類 | 意義 | 交易動作 |
|---|---|---|
| 🔵 **趨勢雷達（反轉）** | 接近趨勢切換點，變盤初期 | 提早佈局 |
| 🟡 **蓄勢待發（回檔）** | 回檔整理中 | 等拉回上車 |
| 🟢 **趨勢穩健** | 已進入上升軌道、動能延續 | 追蹤中波段續抱 |

**每檔股票必備欄位：**
- `Ticker`
- `板塊/產業別`
- `Core Score`（綜合強度分數，用於同板塊內排名）
- `反轉點`（訊號觸發日 / 距今天數）
- **`趨勢變化後漲跌幅`**（訊號出現當下價格 → 目前價格的報酬率，讓使用者一眼看出「訊號準不準、跟了賺多少」）

**核心互動：**
- 依板塊篩選 → 該板塊內三種狀態各自排行前 N 名（同板塊內比較，找出「同學裡最強的那個」）
- 點 ticker 進個股頁：看分數、反轉點、反轉難度，並可切 6個月/1年檢視趨勢區間

**API 骨架：**
```
GET /api/sector-trends?sector={id}&mode={reversal|pullback|bullish}
→ ticker, sector, core_score, signal_date, price_at_signal, price_now, change_pct_since_signal
```

## 二、必留：其他判斷「值不值得跟」的獨特訊號模組

以下是市場上少見、能提供其他工具（Finviz/TradingView）沒有的資訊維度，建議保留：

### 1. 內部人交易（Insider Trading）
高管自掏腰包買自家股票 = 最誠實的訊號。
- 近 3/7/30/60 天窗口切換
- 依「買入金額」排序，抓連續買入 / CEO大額買入

### 2. 機構持股異動（13D/13G）
- **狼群效應**：多家機構同時買進同一檔
- **階梯加倉**：同機構連續多次加碼
→ 這兩個偵測邏輯是核心賣點，一定要留

### 3. 趨勢王者（Proven Winners）
歷史上曾出現高績效趨勢訊號的股票，近期同類型訊號重現 → 用歷史勝率做二次篩選，比單純看當下訊號更有把握。

### 4. Trend Core 選股（歷史回測排行）
在真的下單前，回頭驗證「這個策略在這檔股票身上，過去到底賺不賺錢」，含損益與交易明細。這是把「訊號」跟「有沒有用」直接綁在一起，避免看訊號但沒驗證就進場。

## 三、可以先砍掉 / 非必要（效率優先原則下可延後）

- 題材網路類主題頁（AI基建、太空經濟等）— 屬於內容/教育向，非即時交易決策
- 估值儀表板 — 可作為次要濾網，非首要
- 網格策略儀表板 — 屬於特定策略工具，非通用核心
- 期權儀表板 — 除非使用者明確做賣方策略，否則非首要
- 每日訊號 Blog / 深度研究 — 內容行銷用途，非交易工具本體
- 完整導覽列 / 品牌 CTA / 法務頁尾 — 純資訊性，開發優先度最低

## 四、精簡後的頁面骨架

```
[全站搜尋] 輸入代號快速跳轉
   ↓
[板塊選擇器] → 選一個板塊/產業
   ↓
[三欄戰術面板]  反轉雷達 | 蓄勢待發 | 趨勢穩健
   每欄 Top N：Ticker / Core Score / 反轉點 / 訊號後漲跌幅
   ↓
[點擊 Ticker] → 個股詳情：分數走勢、反轉難度、6M/1Y 區間圖
   ↓
[輔助訊號區] 內部人買入 / 機構狼群效應&階梯加倉 / 趨勢王者重現
   ↓
[驗證] 進 Trend Core 選股查歷史回測績效，再決定是否真的進場
```

一句話核心邏輯：**先用板塊+三段式趨勢定位「現在該做什麼」，再用內部人/機構/歷史勝率三個獨特訊號交叉驗證「這個訊號可不可信」，最後用回測數據確認「歷史上到底賺不賺錢」。**
